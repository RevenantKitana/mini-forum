## [1.10.0] - 2026-02-09

### 🐛 Bug Fixes

#### Frontend - React Query Cache Invalidation on Auth Changes
- ✅ **Fix: Cache not invalidating on login/logout causing permission issues**
  - **Problem**: Users could see cached posts from previous user sessions due to React Query not clearing cache on auth state change
  - **Root Cause**: When user A (admin) views restricted posts, then logs out and user B (guest) logs in, React Query returns cached data from user A's session
  - **Solution**: Added `InvalidationHandler` component in `AuthContext` that invalidates all data queries when auth state (user.id, user.role) changes
  - **Implementation**: 
    - Added import for `useQueryClient` from React Query
    - Created `InvalidationHandler` component inside `AuthProvider` to monitor user changes
    - Invalidates these query keys on auth change: `posts`, `search`, `bookmarks`, `comments`, `categories`
    - Triggers on both login and logout to ensure fresh data retrieval
  - This fix ensures permission-based content (viewPermission filters) is correctly applied after any auth state change
  - Files: `frontend/src/contexts/AuthContext.tsx`

#### Backend - Category viewPermission Filtering for All Content
- ✅ **Fix viewPermission Filtering on Posts & Comments**: Sửa lỗi khách có thể xem bài viết/comment/bookmark từ danh mục private
  - **Post Service** - Thêm hàm `buildViewPermissionFilter()` để filter posts theo category.viewPermission
    - Cập nhật `getPostById()`, `getPostBySlug()`, `getPostsByAuthor()` kiểm tra category permission
    - Cập nhật `getPosts()`, `getFeaturedPosts()`, `getLatestPosts()` (cũ đã fix)
  - **Search Service** - Áp dụng permission filter cho search functionality
    - Cập nhật `searchPosts()` để only show posts from categories users have access to
    - Cập nhật `getSearchSuggestions()` để filter suggestions theo permission
  - **Bookmark Service** - Áp dụng permission filter cho bookmarks
    - Cập nhật `getUserBookmarks()` để only show bookmarks of accessible posts
  - **Comment Service** - Áp dụng permission filter cho comments
    - Cập nhật `getCommentById()` để check category permission trước khi return comment
    - Cập nhật `getCommentReplies()` để check category permission trước khi return replies
  - **Permission Rules**:
    - Khách (guest) chỉ thấy posts/comments/bookmarks từ categories có viewPermission='ALL'
    - Member thấy từ categories có viewPermission='ALL' hoặc 'MEMBER'

#### Issue #4: 429 Rate Limit Error Crashing Frontend
